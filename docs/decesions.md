# 설계 결정 기록 (Decision Log)

## 프로젝트 개요
- 목적: 공공데이터로 대중교통 소외지역을 분석하는 DB 프로젝트.
  실제 초점은 **DB 설계 + 쿼리 성능 튜닝** (2026년 7~8월).
- 타깃: 공기업 우선, 사기업(백엔드) 병행. 8월에 Spring API 추가 예정.

## 기술 스택
- DB: PostgreSQL 17 + PostGIS 3.6
- 백엔드(예정): Spring / Java
- 이유: PostGIS 공간쿼리 + 파티셔닝/윈도우함수 등 PostgreSQL 고유 기능을
  깊게 다뤄 차별화. (주제보다 "실행 깊이"로 차별화하는 전략)

## 범위 (초기)
- 지하철만 먼저 (데이터 깔끔). 버스는 이후 확장.
- 승하차 데이터는 시간(1시간) 단위까지 (튜닝 연습거리 확보).

## 2026-07-06 stop(역) 테이블
- 데이터 출처: 서울교통공사 지하철역 좌표(위경도) CSV.
- 인코딩: 원본이 CP949라 한글 깨짐 → `iconv -f CP949 -t UTF-8`로 UTF-8 변환.
- 파일이 탭이 아니라 콤마 구분(CSV)임을 확인.
- 원본 8컬럼 중 stop_code/name/line/lat/lon만 사용.
  (연번, 작성일자, 작성기준일은 분석에 불필요 → 제외 = 정규화)
- **staging 패턴**: 원본을 stg_stop(전부 text)에 그대로 받고 →
  정제·형변환 후 stop으로 이관. (실무 ETL 방식, 로딩 실패 최소화)
- **geom을 lat/lon과 별도 컬럼**으로 둠: 원본 숫자 보존 +
  공간 인덱스(GiST)용 좌표 분리. ST_MakePoint(lon, lat) 순서 주의.
- 공간 인덱스: `stop_geom_idx` (GiST) — 반경 검색 성능용.

## 인프라 이슈 & 해결
- PostgreSQL이 EDB 18 / Homebrew 14 두 개 충돌 → EDB 18을 중지·자동시작 해제.
- PostGIS가 14용이 아니라 17/18용으로 설치돼 있어, Homebrew 17로 전환해 해결.

## 미해결 / 확인 필요
- 고유역번호(외부역코드)가 환승역에서도 유일한지 확인 (PK 이슈 가능).
- 좌표계는 WGS84(SRID 4326) 사용 — 거리 계산 시 미터 단위 변환(geography 캐스팅) 고려.
